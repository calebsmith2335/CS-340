Q: How do you write programs that are maintainable, readable, and adaptable? Especially consider your work on the CRUD Python module from Project One, which you used to connect the dashboard widgets to the database in Project Two. What were the advantages of working in this way? How else could you use this CRUD Python module in the future?
A: I write programs that are maintainable, readable, and adaptable by organizing code into reusable components, using meaningful variable and function names, and separating different responsibilities into their own modules. In Project One, I developed a CRUD Python module that handled all communication between the application and the MongoDB database. By separating the database operations from the dashboard code in Project Two, I was able to keep the user interface and data access layers independent. This made the code easier to read, troubleshoot, and update.
One advantage of working this way was that the dashboard could retrieve and filter data without needing to know how the database queries were performed. Any changes to the database connection or query logic could be made within the CRUD module without affecting the dashboard components. This modular design improved code reuse and reduced duplication. In the future, the CRUD module could be adapted for other applications that require MongoDB access, such as inventory management systems, customer databases, or other data-driven dashboards.

Q: How do you approach a problem as a computer scientist? Consider how you approached the database or dashboard requirements that Grazioso Salvare requested. How did your approach to this project differ from previous assignments in other courses? What techniques or strategies would you use in the future to create databases to meet other client requests?
A: I approach problems by first identifying the requirements and then breaking the overall problem into smaller, manageable tasks. For the Grazioso Salvare project, I began by understanding the database structure and the information needed by the dashboard. I created the CRUD module to establish database connectivity and then gradually added features to the dashboard, including filtering capabilities, data tables, charts, and geolocation mapping.
Compared to previous assignments, this project required integrating multiple technologies rather than focusing on a single programming concept. Instead of writing standalone programs, I had to consider how the database, Python code, and dashboard components worked together as a complete system. In the future, I would continue using modular design, incremental development, and thorough testing to create databases and applications that can be easily expanded to meet changing client requirements.
